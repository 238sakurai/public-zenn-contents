みなさん、こんにちは。Sakutaro[@saku_238](https://x.com/saku_238) です。  

このブログは[Dress Code Advent Calendar 2025](https://adventar.org/calendars/12017)の6日目の記事です！

僕は2025年12月1日から Dress Code 株式会社でエンジニアとして働き始めました。

直近5年間は技術広報だったのですが、もともとは新卒から10年以上エンジニアをやっていました。
新卒時代はネットワークエンジニアやったり、データエンジニアやったり、 兼務で社内コーポレートエンジニアやったりしていました。
ソースコードを触ってきた世界に5年ぶりに戻ってきたリスタート組です。

今回の記事では12月1日に入社した1週間の体験記録をご紹介します。
入社1週間（12/1 月〜12/5 金）で感じたことを、未来の自分に向けて残しておきます。

## 1週間のスケジュール
5日間のスケジュールはざっくりこんな感じでした（詳細は割愛しますが、だいたいの流れだけ紹介します）。

* 12/1(月) : 全社挨拶・オンボーディング
* 12/2(火) : PCセットアップ・社内資料の熟読
* 12/3(水) : 会議・最初のタスクにアサイン
* 12/4(木) : 会議・タスク前のチュートリアル
* 12/5(金) : 会議・業務着手・OKR確認


それでは、ここからは各日についてもう少し詳しく書いていきます。

### 1日目 (12/1(月)) : 全社挨拶・オンボーディング

![PCセットアップの様子](/images/articles/dresscode-2025-12-06_adventcalendar/2025-12-01_09-49-34_339.jpeg)


オフィスに入室すると僕の固定席があり、僕専用のMacBookProM4と入館証IDと名刺がありました。
それだけでちょっと泣きそうになりました（大げさじゃなく）。

全員の前で挨拶して、そこからランチまでがっつりオンボーディング、エンジニアチーム全員ガチ中華屋でランチ、その後15時までオンボーディングをしました。 


PCセットアップではIDE を開いた瞬間に、**指が勝手にショートカットを押せている**と気づき、ターミナルで `npm install` を眺めながら、「ここからまた積み上げていこう」と静かに決意した1日でした。



## 2日目：Git と GitHub に帰還 → Zenn公開までいってしまう


事業理解、技術理解、ツール理解…三位一体で攻めたら脳がガッツリ疲れた。

## 3日目：プロダクトの呼吸を知る

3日目からは、プロダクトの全体像を掴みにいくフェーズ。

今日は業務理解として、各種資料のinput

GoogleDrive
→ Gemini ・Deep Researchに聞け!
（要件・会議資料）Notion
→ NotionAIに聞け!
ソースコード・yaml
→ cursorに聞け!

AIが欲しい情報を探してくれて、エンジニア復帰余裕じゃん!!って思っていた矢先にタスクがきた。

そこで気づいたのはAIが強いだけだけで、僕はまだレベル1でした!!


## 4日目：初めての機能追加とペアプロ

今日は会議が多めな1日。
- チームでのTips共有
- アーキテクチャ方針についてのディスカッション
- PMFやValue Designの考え方も学び中

会議を終えて、CI/CDのチュートリアルを進めていたら  
サンプルコードに any 型が紛れ込んでて、pre-commitで殴られた🥊  
（lefthookは容赦ないし、ﾅﾆﾓﾜｶﾗﾝ）

## 5日目：チームに混ざっていく感覚:WIP

Slack 上でも、**「質問ウェルカム」な雰囲気**がはっきりしていて、  
遠慮せずに聞ける環境がどれだけ心強いかを改めて実感しました。

1週間を終えた時点で、

- コードを書く楽しさを思い出せた
- チームとしてのリズムも少しずつ掴めてきた
- 「ここからこうなりたい」という具体的なイメージが湧いてきた

そんな気持ちで、いい意味で心地よい疲れを感じながら金曜日を終えました。

## おわりに：リスタート組としてのこれから

この1週間は、「転生したらエンジニアだった」というタイトルどおり、  
**エンジニアとしての自分をもう一度インストールし直す時間**でした。

もちろん、まだまだ追いつけていないところだらけですが、  
前職での経験（技術広報・発信・言語化）が、  
思った以上にエンジニアとしても活かせそうだな、という手応えもあります。

ちなみに、そのときどきの細かい感情やメモは X にも流していました。  

これから少しずつ、具体的な技術の話もアウトプットしていければと思います。  
まずは 1 週間、無事にエンジニアとして再スタートを切れた記録でした。
